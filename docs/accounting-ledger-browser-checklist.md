# Accounting Ledger 브라우저 수동 체크리스트

> 개정 2026-07-11 · 대상 앱 버전 `0.17`

이 문서는 하네스의 `browser-roundtrip`(MANUAL) 게이트와 `CLAUDE.md` 검증 절차 6단계가 가리키는 **반복 실행용 시나리오 목록**이다. 자동 브라우저 러너가 도입되기 전까지, `index.html`을 실제로 바꾼 릴리스에서는 아래를 데스크톱과 모바일 폭에서 실행하고 결과(통과/실패/미확인)를 `docs/accounting-ledger-harness-baseline.md`에 남긴다.

실행 방법: `index.html`을 브라우저로 연다(정적 파일이라 서버 불필요). 운영 확인은 `https://hanwha27-tdtu.github.io/Accounting_ledger/`에서 한다. 개발자 도구 콘솔을 열어 둔다.

## 공통 기준

- [ ] 콘솔 error 0건(경고는 사유 확인).
- [ ] 데스크톱(예: 1280px)과 모바일(예: 390x844) 모두 가로 overflow 없음.
- [ ] 긴 한국어 문구·URL이 잘리거나 겹치지 않음.

## 1. 로드와 오프라인 (0.01)

- [ ] 첫 로드시 대시보드가 뜨고, 사업자 미설정 안내가 보인다.
- [ ] 네트워크를 끊어도(오프라인) 거래 입력·저장이 되고, 상단 상태가 `오프라인 · 로컬 저장`으로 바뀐다.
- [ ] 다시 온라인이 되면 상태가 복구된다.

## 2. 사업자 설정과 거래·전표 (0.01)

- [ ] 설정에서 사업자 정보를 저장하면 상단 컨텍스트와 대시보드 안내가 갱신된다.
- [ ] 거래(수입/비용/자산) 입력 시 공급가액·부가세·합계 미리보기가 맞다.
- [ ] 저장 후 전표 검토에서 차변 합계 = 대변 합계(불균형이면 저장 차단).
- [ ] 장부 화면의 수입/비용/자산 열과 상태 배지가 맞다.

## 2b. 국세청 업종코드 검색 선택 (0.11)

- [ ] 설정 → 국세청 업종코드에서 "경영 컨설팅" 검색 시 `741400 경영 컨설팅업` 등이 뜬다.
- [ ] "의원" 검색 시 `851201 일반의원` 등이 뜬다. 숫자 "741" 검색 시 해당 코드들이 뜬다.
- [ ] 결과를 클릭하면 코드(6자리)와 업종명이 자동으로 채워지고 "선택됨"에 표시된다.
- [ ] 후보 안내와 홈택스 공식 조회·경비율 고시 링크가 보이고 새 탭으로 열린다.
- [ ] 코드를 직접 입력하는 fallback도 여전히 가능하다. 저장 후 다시 열면 선택값이 유지된다.
- [ ] (0.13) "940600"이나 "자문" 검색으로 인적용역 코드를 고르면 "인적용역 · 사업자등록번호 없을 수 있음" 힌트가 뜬다.
- [ ] (0.13) 사업자등록번호를 비워두고도 사업자 정보가 저장된다(라벨에 `(선택)`, 안내 문구 표시).
- [ ] (0.14) 사업자등록번호에 틀린 숫자(예: 2208162518)를 넣으면 "검증 실패" 오류가 뜨고, 유효한 번호(2208162517)는 통과한다.
- [ ] (0.14) 로그인 상태에서 대표자명을 비우면 Google 이름이 자동으로 채워진다(수정 가능).
- [ ] (0.14) 업종코드를 검색·선택하면 큰 업종이 자동으로 설정된다(예: 741400 경영컨설팅 → 자문·전문직, 940xxx → 프리랜서).
- [ ] (0.15) 업종코드를 선택하면 경비율이 자동 표시된다(예: 741400 → 단순 77.3% · 기준 23.1%; 940903 → 초과율 46.4% 함께). "후보·귀속연도·출처" 표기 확인.
- [ ] (0.16) 업종코드를 선택하면 기장의무 기준이 자동 표시된다(예: 851201 의료 → 7,500만원, 501101 도소매 → 3억원, 151101 제조 → 1.5억원). "소득세법 시행령 §208⑤" 근거 표기 확인.
- [ ] (0.16) 부가세 면세 가능성이 근거 조문과 함께 후보로 표시된다(예: 851201 → 의료보건 용역 §26①5호 · 미용 등 일부 과세 · 개별 판단; 741400 컨설팅 → 면세 표시 없음). "개별 판단" 문구 확인.

## 3. 연결 진단과 로그인 (0.02–0.03)

- [ ] 설정 → 연결 진단: Data API `정상`, 익명 회계자료 `차단 정상`, Google OAuth 상태 표시.
- [ ] 가이드 → 구글클라우드 연결방법: 값이 현재 설정(APP_INFO/GuideService)과 일치하고 복사가 된다.
- [ ] owner(`hanwha27@gmail.com`) 로그인 후 상단이 `동기화 완료`로 바뀐다. 비허용 계정은 차단된다.

## 4. businesses CRUD·RLS 왕복 (0.04)

- [ ] 로그인 상태에서 설정 → `사업자 CRUD·RLS 왕복 검증` 실행 시 생성·조회·수정·소프트삭제 격리·정리 5단계가 모두 `통과`.
- [ ] 검증 후 클라우드 `businesses`에 `__rls_probe__` 잔존 행이 없다.
- [ ] 비로그인 상태에서는 이 패널이 보이지 않는다.

## 5. 데이터 관리 (0.05)

- [ ] 관리 → 데이터 관리: 6개 카드가 2열 대칭으로 보이고, 모바일에서 1열로 접힌다.
- [ ] JSON 백업 → 파일 생성. 복원 → 확인 후 로컬 데이터가 파일 기준으로 교체된다.
- [ ] 부분 삭제: 거래 1건 선택·삭제 시 목록에서 사라지고, 전표·라인도 함께 소프트삭제된다.
- [ ] 일괄 삭제: 이중 확인 후 로컬 장부가 비고, Supabase 연결 설정은 유지된다.

## 6. 다기기 삭제 수렴 — 좀비 방지 (0.06)

기기 A, B 두 브라우저(또는 프로필)로 같은 owner 로그인:

- [ ] A에서 거래를 만들고 동기화 → B에서 동기화 시 그 거래가 보인다.
- [ ] A에서 그 거래를 삭제하고 동기화 → B에서 동기화 시 **B에서도 사라진다**.
- [ ] B에서 다시 동기화해도 삭제된 거래가 되살아나지 않는다(멱등).
- [ ] 삭제한 A에서도 재동기화 시 그 거래가 되돌아오지 않는다.

### 최종본 지정 (0.08, owner 전용)

- [ ] owner로 로그인하면 데이터 관리 동기화 카드에 `이 기기를 최종본으로` 버튼이 보인다(비-owner에겐 없음).
- [ ] B에 아직 동기화하지 않은 로컬 거래가 있는 상태에서 A가 `이 기기를 최종본으로`를 실행한다(확인창 뜸).
- [ ] A 실행 후 canonical 표시가 올라간다.
- [ ] B에서 동기화하면 B의 로컬 전용 미동기화 거래가 사라지고 A(클라우드) 기준으로 맞춰지며 canonical이 같아진다.

### 빈 클라우드 가드 (0.10)

- [ ] 로컬에 사업자·거래가 있는 상태에서, 클라우드가 회계자료를 못 주는 상황(예: 잘못된 로그인/권한)에 최종본 적용이 실행돼도 **로컬 데이터가 지워지지 않고** "로컬 데이터를 지키려고 최종본 적용을 중단했습니다" 안내가 뜬다.

## 7. 허용 사용자 관리 — owner 전용 (0.07)

owner(`hanwha27@gmail.com`)로 로그인:

- [ ] 설정에 `허용 사용자 관리(owner 전용)` 패널이 보이고, 현재 허용 목록이 표시된다.
- [ ] 이메일·역할·메모로 사용자를 추가하면 목록에 `활성`으로 나타난다.
- [ ] `차단`을 누르면 상태가 `차단`으로 바뀌고, `허용`으로 되돌릴 수 있다.
- [ ] 소유자 자기 행은 `보호됨`으로 차단 버튼이 없다.
- [ ] 잘못된 이메일·중복 이메일 추가 시 명확한 오류 메시지가 뜬다.
- [ ] owner가 아닌 계정으로 로그인하면 이 패널이 보이지 않는다.
- [ ] 차단(blocked)된 계정으로 로그인하면 회계자료에 접근하지 못하고 차단된다.

## 8. Cloudinary 증빙 첨부 (0.09) — 사용자 Cloudinary 설정 필요

사전: Cloudinary 계정에서 제한된 **unsigned upload preset**(허용 형식 jpg·png·pdf, 폴더·최대 크기 제한) 생성.

- [ ] 증빙 화면: Cloudinary 미설정 시 설정 안내 배너와 `설정 열기`가 보인다.
- [ ] 설정 → Cloudinary 증빙 저장소에 cloud name과 unsigned preset을 저장하면 배지가 `설정됨`으로 바뀐다.
- [ ] 비밀키 형태(`cloudinary://`로 시작하는 API 환경변수 URL이나 secret 값)를 넣으면 저장이 거부된다.
- [ ] 거래의 `파일 첨부`로 이미지 업로드 시 썸네일이 뜨고 배지가 `1개 첨부`로 바뀐다.
- [ ] PDF 업로드 시 파일 링크가 뜨고 클릭하면 원본이 열린다.
- [ ] 오프라인에서 첨부해도 로컬에 저장되고, 로그인·동기화 후 클라우드(evidence_files)에 반영된다.
- [ ] 15MB 초과 파일은 첨부가 거부된다.
- [ ] 잘못된 preset이면 명확한 오류 메시지가 뜬다.
- [ ] (0.12) 첨부된 증빙의 × 버튼으로 제거하면 목록에서 사라지고, 그 거래에 다른 증빙이 없으면 `미첨부`로 돌아간다. 두 기기면 동기화 후 다른 기기에서도 사라진다.

## 8b. 간편장부 계정과목 분류표 (0.17)

- [ ] 가져오기 화면에 `간편장부 계정과목 분류표` 패널이 보이고, 수입·비용·제조비용·자산 4개 그룹으로 25개 계정과목과 공식 설명이 나온다.
- [ ] 각 그룹 머리말에 `간편장부 수입금액/비용/자산 열` 대응이 표시된다(수입→수입금액, 비용·제조비용→비용, 자산 매입→자산).
- [ ] 데스크톱·모바일 모두 표가 가로 overflow 없이 접힌다(설명이 길어도 줄바꿈).

## 9. 개발 기록·가이드·접근성

- [ ] 개발 기록: 앱 버전, 저장소·동기화 상태(CRUD 왕복 포함), Guardian 레지스트리가 보인다.
- [ ] 키보드 이동, 버튼 라벨, 터치 크기, 색 대비가 무리 없다.
