# Accounting Ledger 브라우저 수동 체크리스트

> 개정 2026-07-12 · 대상 앱 버전 `0.33`

이 문서는 하네스의 `browser-roundtrip`(MANUAL) 게이트와 `CLAUDE.md` 검증 절차 6단계가 가리키는 **반복 실행용 시나리오 목록**이다. 자동 브라우저 러너가 도입되기 전까지, `index.html`을 실제로 바꾼 릴리스에서는 아래를 데스크톱과 모바일 폭에서 실행하고 결과(통과/실패/미확인)를 `docs/accounting-ledger-harness-baseline.md`에 남긴다.

실행 방법: `index.html`을 브라우저로 연다(정적 파일이라 서버 불필요). 운영 확인은 `https://hanwha27-tdtu.github.io/Accounting_ledger/`에서 한다. 개발자 도구 콘솔을 열어 둔다.

## 공통 기준

- [ ] 콘솔 error 0건(경고는 사유 확인).
- [ ] 데스크톱(예: 1280px)과 모바일(예: 390x844) 모두 가로 overflow 없음.

## 1. 로드와 오프라인 (0.01)

- [ ] 첫 로드시 대시보드가 뜨고, 사업자 미설정 안내가 보인다.

## 2. 사업자 설정과 거래·전표 (0.01)

- [ ] 설정에서 사업자 정보를 저장하면 상단 컨텍스트와 대시보드 안내가 갱신된다.

## 2c. 해외 거래 입력 (0.28)

- [x] 거래 입력에서 `해외 거래` 체크 시 통화·외화 금액·환율 칸이 나타나고, 외화×환율이 총 금액에 자동으로 채워진다(공급가액·부가세 미리보기도 갱신). **헤드리스 검증: 100×1,350 → 135,000원.**
- [x] 과세유형이 `면세사업`인 사업자가 비용/자산으로 해외 거래를 입력하면 **대리납부 주의(§52)** 경고가 뜬다. **헤드리스 검증됨.**
- [x] (0.29) 해외 거래를 저장하면 통화·외화 금액·환율·해외 여부가 **구조화된 값으로 거래에 저장**되고, 장부 거래내용에 `🌐 해외` 배지가 붙는다(마우스오버로 통화·환율). **헤드리스 검증: is_overseas=true·USD·100·1350 저장·배지 표시.** 초기화 시 해외 칸이 접힌다.
- [x] (0.30) 통화 칸에서 목록(datalist)으로 통화를 고를 수 있다 — 기축·주요 통화(USD·EUR·JPY·GBP·CNY 등)와 **UZS 우즈베키스탄 숨** 포함(28개). 목록에 없는 통화도 3글자 코드로 직접 입력된다. **헤드리스 검증: 28개 옵션·UZS 존재.**

## 6. 다기기 삭제 수렴 — 좀비 방지 (0.06)

기기 A, B 두 브라우저(또는 프로필)로 같은 owner 로그인:

- [ ] A에서 거래를 만들고 동기화 → B에서 동기화 시 그 거래가 보인다.
- [ ] A에서 그 거래를 삭제하고 동기화 → B에서 동기화 시 **B에서도 사라진다**.
- [ ] B에서 다시 동기화해도 삭제된 거래가 되살아나지 않는다(멱등).
- [ ] 삭제한 A에서도 재동기화 시 그 거래가 되돌아오지 않는다.

### 최종본 지정 (0.08, owner 전용)

- [ ] owner로 로그인하면 데이터 관리 동기화 카드에 `이 기기를 최종본으로` 버튼이 보인다(비-owner에겐 없음).
- [ ] B에 아직 동기화하지 않은 로컬 거래가 있는 상태에서 A가 `이 기기를 최종본으로`를 실행한다(확인창 뜸).
- [ ] A 실행 후 canonical 표시가 올라간다.
- [ ] B에서 동기화하면 B의 로컬 전용 미동기화 거래가 사라지고 A(클라우드) 기준으로 맞춰지며 canonical이 같아진다.

### 빈 클라우드 가드 (0.10)

- [ ] 로컬에 사업자·거래가 있는 상태에서, 클라우드가 회계자료를 못 주는 상황(예: 잘못된 로그인/권한)에 최종본 적용이 실행돼도 **로컬 데이터가 지워지지 않고** "로컬 데이터를 지키려고 최종본 적용을 중단했습니다" 안내가 뜬다.

## 8. Cloudinary 증빙 첨부 (0.09) — 사용자 Cloudinary 설정 필요

사전: Cloudinary 계정에서 제한된 **unsigned upload preset**(허용 형식 jpg·png·pdf, 폴더·최대 크기 제한) 생성.

- [ ] 증빙 화면: Cloudinary 미설정 시 설정 안내 배너와 `설정 열기`가 보인다.
- [ ] 설정 → Cloudinary 증빙 저장소에 cloud name과 unsigned preset을 저장하면 배지가 `설정됨`으로 바뀐다.
- [ ] 비밀키 형태(`cloudinary://`로 시작하는 API 환경변수 URL이나 secret 값)를 넣으면 저장이 거부된다.
- [ ] 거래의 `파일 첨부`로 이미지 업로드 시 썸네일이 뜨고 배지가 `1개 첨부`로 바뀐다.
- [ ] PDF 업로드 시 파일 링크가 뜨고 클릭하면 원본이 열린다.
- [ ] 오프라인에서 첨부해도 로컬에 저장되고, 로그인·동기화 후 클라우드(evidence_files)에 반영된다.
- [ ] 15MB 초과 파일은 첨부가 거부된다.
- [ ] 잘못된 preset이면 명확한 오류 메시지가 뜬다.
- [ ] (0.12) 첨부된 증빙의 × 버튼으로 제거하면 목록에서 사라지고, 그 거래에 다른 증빙이 없으면 `미첨부`로 돌아간다. 두 기기면 동기화 후 다른 기기에서도 사라진다.

## 8d. 가이드 법령 안내 버튼 (0.20)

- [ ] 가이드 화면에 버튼 4개(구글클라우드 연결 + 기장의무 판단 + 부가가치세 면세 + 추계신고와 경비율)가 보인다.
- [ ] 각 법령 버튼을 누르면 쉬운 설명 + 기준금액 + `근거` 블록(조문)이 나온다. **헤드리스로 근거 조문 렌더 검증됨**(기장의무=소득세법 §160·시행령 §208⑤; 추계=소득세법 §80③·시행령 §143; 면세=부가가치세법 §26①).
- [ ] 기장의무·추계 버튼의 `설정에서 업종코드 고르기`/`리포트에서 계산기 열기` 링크가 해당 화면으로 이동한다.
- [x] (0.21) 각 법령 버튼에 `왜 이렇게 정했을까요? (눈높이 설명)` 블록이 목적·취지와 함께 보인다. **헤드리스로 3개 토픽 모두 렌더 검증됨**(기장의무·면세·추계).

## 8e. 간편장부 Excel 미리보기 (0.24)

- [x] 가져오기 화면에서 국세청 간편장부(.xlsx)를 선택하면 「장부」 시트를 읽어 **읽은 건수·귀속연도·수입/비용/자산 건수·합계**가 뜬다. **실제 사용자 샘플(945건, 2025년, 수입 40,000,000·비용 38,625,037)로 헤드리스 브라우저 검증 완료.**

- [ ] 데스크톱·모바일 모두 미리보기 표가 가로 스크롤 컨테이너 안에서만 넘친다(본문 overflow 없음).
- [x] (0.25) `장부에 반영` 버튼을 누르면 확인 후 거래가 복식부기 원장에 기록되고 장부 화면으로 이동한다. **실 샘플로 헤드리스 검증: 945건 전기·차변=대변 81,651,988(전부 균형)·계정과목 자동생성 6개.**
- [x] (0.25) 같은 파일을 다시 반영해도 **중복이 쌓이지 않는다**(내용 기반 결정적 id). 헤드리스 재반영 검증: 거래 수 945 유지, 균형 유지.
- [ ] (0.25) 사업자 미설정 시 반영 버튼이 비활성이고 설정 안내가 뜬다. 없던 계정과목·거래처는 자동 생성된다(장부에 이름 그대로 표시).

- [x] (0.26) `일괄 적용` 후 경고 512→0, 반영 시 945건 균형 유지(차변=대변). 오류 행은 반영에서 자동 제외. **헤드리스 검증 완료.**
- [ ] (0.26) 모달의 닫기/X로 닫히고, 인라인 `검증·반영 창 열기` 버튼으로 다시 열린다. 데스크톱·모바일 모두 모달이 화면 안에 들어온다.
