# Accounting Ledger 브라우저 수동 체크리스트

> 개정 2026-07-15 · 대상 앱 버전 `0.46`

이 문서는 하네스의 `browser-roundtrip`(MANUAL) 게이트와 `CLAUDE.md` 검증 절차 6단계가 가리키는 **반복 실행용 시나리오 목록**이다. 자동 브라우저 러너가 도입되기 전까지, `index.html`을 실제로 바꾼 릴리스에서는 아래를 데스크톱과 모바일 폭에서 실행하고 결과(통과/실패/미확인)를 `docs/accounting-ledger-harness-baseline.md`에 남긴다.

실행 방법: `index.html`을 브라우저로 연다(정적 파일이라 서버 불필요). 운영 확인은 `https://hanwha27-tdtu.github.io/Accounting_ledger/`에서 한다. 개발자 도구 콘솔을 열어 둔다.

## 공통 기준

- [ ] 콘솔 error 0건(경고는 사유 확인).
- [ ] 데스크톱(예: 1280px)과 모바일(예: 390x844) 모두 가로 overflow 없음.
- [ ] 긴 한국어 문구·URL이 잘리거나 겹치지 않음.

## 1. 로드와 오프라인 (0.01)

- [ ] 첫 로드시 대시보드가 뜨고, 사업자 미설정 안내가 보인다.
- [ ] 네트워크를 끊어도(오프라인) 거래 입력·저장이 되고, 상단 상태가 `오프라인 · 로컬 저장`으로 바뀐다.
- [ ] 다시 온라인이 되면 상태가 복구된다.

## 2. 사업자 설정과 거래·전표 (0.01)

- [ ] 설정에서 사업자 정보를 저장하면 상단 컨텍스트와 대시보드 안내가 갱신된다.
- [ ] 거래(수입/비용/자산) 입력 시 공급가액·부가세·합계 미리보기가 맞다.
- [ ] 저장 후 전표 검토에서 차변 합계 = 대변 합계(불균형이면 저장 차단).
- [ ] 장부 화면의 수입/비용/자산 열과 상태 배지가 맞다.

## 2b. 국세청 업종코드 검색 선택 (0.11)

- [ ] 설정 → 국세청 업종코드에서 "경영 컨설팅" 검색 시 `741400 경영 컨설팅업` 등이 뜬다.
- [ ] "의원" 검색 시 `851201 일반의원` 등이 뜬다. 숫자 "741" 검색 시 해당 코드들이 뜬다.
- [ ] 결과를 클릭하면 코드(6자리)와 업종명이 자동으로 채워지고 "선택됨"에 표시된다.
- [ ] 후보 안내와 홈택스 공식 조회·경비율 고시 링크가 보이고 새 탭으로 열린다.
- [ ] 코드를 직접 입력하는 fallback도 여전히 가능하다. 저장 후 다시 열면 선택값이 유지된다.
- [ ] (0.13) "940600"이나 "자문" 검색으로 인적용역 코드를 고르면 "인적용역 · 사업자등록번호 없을 수 있음" 힌트가 뜬다.
- [ ] (0.13) 사업자등록번호를 비워두고도 사업자 정보가 저장된다(라벨에 `(선택)`, 안내 문구 표시).
- [ ] (0.14) 사업자등록번호에 틀린 숫자(예: 2208162518)를 넣으면 "검증 실패" 오류가 뜨고, 유효한 번호(2208162517)는 통과한다.
- [ ] (0.14) 로그인 상태에서 대표자명을 비우면 Google 이름이 자동으로 채워진다(수정 가능).
- [ ] (0.14) 업종코드를 검색·선택하면 큰 업종이 자동으로 설정된다(예: 741400 경영컨설팅 → 자문·전문직, 940xxx → 프리랜서).
- [ ] (0.15) 업종코드를 선택하면 경비율이 자동 표시된다(예: 741400 → 단순 77.3% · 기준 23.1%; 940903 → 초과율 46.4% 함께). "후보·귀속연도·출처" 표기 확인.
- [ ] (0.16) 업종코드를 선택하면 기장의무 기준이 자동 표시된다(예: 851201 의료 → 7,500만원, 501101 도소매 → 3억원, 151101 제조 → 1.5억원). "소득세법 시행령 §208⑤" 근거 표기 확인.
- [ ] (0.16) 부가세 면세 가능성이 근거 조문과 함께 후보로 표시된다(예: 851201 → 의료보건 용역 §26①5호 · 미용 등 일부 과세 · 개별 판단; 741400 컨설팅 → 면세 표시 없음). "개별 판단" 문구 확인.

## 2c. 해외 거래 입력 (0.28)

- [x] 거래 입력에서 `해외 거래` 체크 시 통화·외화 금액·환율 칸이 나타나고, 외화×환율이 총 금액에 자동으로 채워진다(공급가액·부가세 미리보기도 갱신). **헤드리스 검증: 100×1,350 → 135,000원.**
- [x] 과세유형이 `면세사업`인 사업자가 비용/자산으로 해외 거래를 입력하면 **대리납부 주의(§52)** 경고가 뜬다. **헤드리스 검증됨.**
- [x] (0.29) 해외 거래를 저장하면 통화·외화 금액·환율·해외 여부가 **구조화된 값으로 거래에 저장**되고, 장부 거래내용에 `🌐 해외` 배지가 붙는다(마우스오버로 통화·환율). **헤드리스 검증: is_overseas=true·USD·100·1350 저장·배지 표시.** 초기화 시 해외 칸이 접힌다.
- [x] (0.30) 통화 칸에서 목록(datalist)으로 통화를 고를 수 있다 — 기축·주요 통화(USD·EUR·JPY·GBP·CNY 등)와 **UZS 우즈베키스탄 숨** 포함(28개). 목록에 없는 통화도 3글자 코드로 직접 입력된다. **헤드리스 검증: 28개 옵션·UZS 존재.**

- [ ] 설정 → 연결 진단: Data API `정상`, 익명 회계자료 `차단 정상`, Google OAuth 상태 표시.

- [ ] owner(`hanwha27@gmail.com`) 로그인 후 상단이 `동기화 완료`로 바뀐다. 비허용 계정은 차단된다.

## 4. businesses CRUD·RLS 왕복 (0.04)

- [ ] 로그인 상태에서 설정 → `사업자 CRUD·RLS 왕복 검증` 실행 시 생성·조회·수정·소프트삭제 격리·정리 5단계가 모두 `통과`.
- [ ] 검증 후 클라우드 `businesses`에 `__rls_probe__` 잔존 행이 없다.
- [ ] 비로그인 상태에서는 이 패널이 보이지 않는다.

## 5. 데이터 관리 (0.05, 0.39에서 카드 구성 갱신)

- [ ] 관리 → 데이터 관리: 카드가 2열 대칭으로 보이고, 모바일에서 1열로 접힌다.
- [ ] 상단 안내 문구가 "정본은 클라우드, 이 기기는 보조 캐시"라는 취지로 보인다.
- [ ] 이 기기 백업 → 파일 생성(파일명에 `-local-` 포함). 복원("이 기기만 복원") → 확인 후 로컬 데이터가 파일 기준으로 교체된다.
- [ ] 부분 삭제: 거래 1건 선택·삭제 시 목록에서 사라지고, 전표·라인도 함께 소프트삭제된다.
- [ ] 일괄 삭제: 이중 확인 후 로컬 장부가 비고, Supabase 연결 설정은 유지된다.

## 5f. 클라우드가 정본 — 백업·복원·강제 새로고침 (0.39)

owner로 로그인한 상태에서 확인(비로그인 시 관련 버튼은 비활성/숨김 처리만 확인):

- [ ] "클라우드에서 백업" 클릭 시 파일이 생성되고(파일명에 `-cloud-` 포함), 이 기기 캐시가 아니라 클라우드 값을 그대로 담고 있다(다른 기기에서 만든 거래도 포함되는지 확인).
- [ ] "클라우드 기준으로 새로고침" 클릭 시 확인창에 "아직 클라우드에 안 올라간 변경이 사라집니다" 경고가 대기 건수와 함께 뜨고(대기가 있을 때만), 실행 후 이 기기 화면이 클라우드 값과 정확히 같아진다.
- [ ] "이 기기만 복원"으로 오래된 백업을 복원한 뒤 잠시 기다리면(자동 동기화), 클라우드 값이 다시 우선해 화면이 원래대로 돌아올 수 있다 — 카드 설명 문구가 이 사실을 미리 안내하는지 확인.
- [ ] "복원 후 모든 기기에 반영"(owner 전용)으로 복원하면, 확인창에 "다른 모든 기기는 이 내용으로 맞춰지고 각 기기 로컬 전용 변경은 버려진다"는 경고가 뜨고, 성공 시 canonical 버전이 올라간다. 다른 기기(B)에서 다음 동기화 시 이 내용으로 실제로 맞춰지는지 확인.
- [ ] 클라우드에서 만든 백업 파일을 "이 기기만 복원"으로 복원해도 이 기기의 동기화 대기 건수(sync_queue)가 0으로 초기화되지 않는다(클라우드 백업엔 없는 로컬 전용 항목이라 보존돼야 함).

## 5b. 기기간 자동 동기화 — 기본값 (0.35)

기기 A, B 두 브라우저(또는 프로필)로 같은 owner 로그인:

- [ ] B에 로그인만 하고(동기화 버튼을 누르지 않음) 잠시 기다리면, A에서 만든 거래가 B에도 자동으로 나타난다.
- [ ] A에서 거래를 저장한 뒤 다른 탭으로 갔다가 다시 이 탭으로 돌아오면(포커스 복귀) 상단 동기화 배지가 자동으로 갱신된다(버튼을 누르지 않아도).
- [ ] 오프라인으로 전환 후 거래를 저장하고, 다시 온라인으로 전환하면 "자동으로 동기화합니다" 안내 후 자동으로 반영된다(버튼 클릭 불필요).
- [ ] 거래 입력 화면에서 입력 중(제출 전)에 백그라운드 자동 동기화가 발생해도 입력하던 값이 사라지지 않는다(다른 기기에서 동시에 거래를 저장해 동기화를 유발해 확인).
- [ ] 설정 화면의 폼(사업자 정보 등)에 입력 중일 때도 마찬가지로 값이 사라지지 않는다.
- [x] 마감 화면에서 마감 사유(`메모`)를 입력 중일 때도 값이 사라지지 않는다(0.40에서 `AUTO_SYNC_UNSAFE_ROUTES`에 `closing` 누락을 발견·수정 — 로그인 없이도 재현 가능한 `hashchange` 강제 재렌더 방식으로 **헤드리스 검증됨**).
- [ ] 데이터 관리 → 클라우드 동기화 카드 문구가 "자동으로 동기화됩니다"로 보이고, `지금 동기화` 버튼은 여전히 동작한다(즉시 반영용).
- [ ] 콘솔에 자동 동기화로 인한 에러가 없다(로그인하지 않은 상태에서 포커스·온라인 이벤트가 발생해도 조용히 무시된다).

## 5c. 가계부 여러 개 관리 (0.36)

- [ ] 설정에 "가계부 관리" 패널이 보이고, 현재 가계부에 "사용 중" 배지가 붙는다.
- [ ] "새 가계부 추가"로 이름·종류(개인/사업자)를 정해 만들면 즉시 그 가계부로 전환되고(topbar 이름이 바뀜), 거래·계정과목이 0건인 새 상태로 시작한다.
- [ ] 가계부 목록에서 "전환" 버튼을 누르면 다른 가계부로 바뀌고, 장부·리포트·대시보드가 그 가계부 자료만 보여준다(다른 가계부의 거래가 섞여 보이지 않는다).
- [ ] 개인 가계부와 사업자 가계부를 하나씩 만들어 오가도 서로의 모드(세무 화면 노출 여부)가 뒤섞이지 않는다.
- [ ] 클라우드 동기화 후 다른 기기에서도 같은 가계부 목록이 보인다(단, "지금 보고 있는" 가계부는 기기별로 다를 수 있다 — 정상 동작).
- [ ] 0.35 이전부터 쓰던 기존 가계부(가계부가 하나뿐이던 설치)는 업그레이드 후에도 그대로 활성 가계부로 유지되고, 모드(개인/사업자)도 그대로 보인다.

## 5d. 마감·마감해제 (0.37)

- [ ] 개인 가계부 모드에서도 왼쪽 메뉴에 "마감"이 보인다(0.37부터 사업자 전용이 아님).
- [ ] 이번 달을 마감하면 목록에 "마감됨" 배지가 뜨고, 몇 건이 있는지 표시된다.
- [ ] 마감한 달 안의 날짜로 새 거래를 저장하려 하면 거래일 칸에 "이 날짜는 마감된 기간입니다..." 오류가 뜨고 저장되지 않는다.
- [ ] 마감하기 전에 이미 있던 거래를(데이터 관리 → 선택 거래 삭제) 지우려 하면 "마감된 기간이라 삭제할 수 없습니다" 안내가 뜨고 실제로 지워지지 않는다.
- [ ] "마감 해제"를 누르면(확인창) 배지가 "열림"으로 바뀌고, 그 즉시 같은 달에 거래를 새로 저장하거나 지울 수 있게 된다.
- [ ] 마감·해제 기록에 날짜(마감일, 마지막 해제일)가 남아 있다 — 마감 이력이 사라지지 않는다.
- [ ] 다른 가계부로 전환하면 마감 목록도 그 가계부 것만 보인다(가계부별로 분리).

## 5e. 가계부 삭제 (0.38)

- [ ] 가계부 관리 목록의 각 가계부에 "삭제" 버튼이 보인다(사용 중인 가계부도 포함).
- [ ] 삭제를 누르면 확인창에 가계부 이름과 "거래·계정과목·거래처·전표·증빙·마감 기록을 삭제합니다"라는 문구가 뜬다.
- [ ] 사용 중이 아닌 가계부를 삭제하면 목록에서 사라지고, 다른 가계부·거래에는 영향이 없다.

## 6. 다기기 삭제 수렴 — 좀비 방지 (0.06)

- [ ] (0.23) 거래 입력 화면의 용어(계정과목·공급가액 등) 밑줄 버튼을 누르면 툴팁이 뜬다. 세법 용어(복식부기·공급가액)는 용어사전과 같은 설명, 앱 용어(부가세·원천거래·전표)는 기존 설명. **용어 해석 경로(용어사전 우선→앱용어 폴백)는 헤드리스로 검증됨**(툴팁 DOM 클릭은 사업자 설정 시 노출되어 수동 확인 대상).

## 8e. 간편장부 Excel 미리보기 (0.24)

- [x] 가져오기 화면에서 국세청 간편장부(.xlsx)를 선택하면 「장부」 시트를 읽어 **읽은 건수·귀속연도·수입/비용/자산 건수·합계**가 뜬다. **실제 사용자 샘플(945건, 2025년, 수입 40,000,000·비용 38,625,037)로 헤드리스 브라우저 검증 완료.**
- [x] 미리보기 표가 일자(월·일 합쳐 YYYY-MM-DD)·계정과목·거래내용·거래처·수입/비용/자산·비고를 원본대로 보여준다(상위 200건). 분류표에 없는 계정과목은 `(미분류)` 표시.
- [ ] 잘못된 파일(장부 시트 없음)이면 명확한 오류가 뜬다.
- [ ] 데스크톱·모바일 모두 미리보기 표가 가로 스크롤 컨테이너 안에서만 넘친다(본문 overflow 없음).
- [x] (0.25) `장부에 반영` 버튼을 누르면 확인 후 거래가 복식부기 원장에 기록되고 장부 화면으로 이동한다. **실 샘플로 헤드리스 검증: 945건 전기·차변=대변 81,651,988(전부 균형)·계정과목 자동생성 6개.**
- [x] (0.25) 같은 파일을 다시 반영해도 **중복이 쌓이지 않는다**(내용 기반 결정적 id). 헤드리스 재반영 검증: 거래 수 945 유지, 균형 유지.
- [ ] (0.25) 사업자 미설정 시 반영 버튼이 비활성이고 설정 안내가 뜬다. 없던 계정과목·거래처는 자동 생성된다(장부에 이름 그대로 표시).
- [x] (0.26) 업로드하면 **검증 모달이 자동으로 열려** 정상/경고/오류 건수를 보여준다. **실 샘플 헤드리스 검증: 정상 945·경고 512(부가세)·오류 0.** 미분류 계정과목은 드롭다운으로 표준 계정과목 치환, 부가세 총액 의심은 `일괄 적용`으로 공급가액 재계산 — 값 변경은 승인해야 반영.
- [x] (0.26) `일괄 적용` 후 경고 512→0, 반영 시 945건 균형 유지(차변=대변). 오류 행은 반영에서 자동 제외. **헤드리스 검증 완료.**
- [ ] (0.26) 모달의 닫기/X로 닫히고, 인라인 `검증·반영 창 열기` 버튼으로 다시 열린다. 데스크톱·모바일 모두 모달이 화면 안에 들어온다.

## 8f. 월별 사용현황 (0.31)

- [x] 상단(모든 화면 공통)에 `월별 사용현황` 버튼이 있다. 사업자 미설정 시 클릭하면 오류 안내만 뜨고 모달은 열리지 않는다. **헤드리스 검증됨.**
- [x] 사업자·거래가 있으면 버튼 클릭 시 모달이 열리고, 계정과목(열) × 월(행) 표에 금액이 정확히 합산되어 보이며 행·열 합계가 맞는다. **헤드리스 검증: 1월 1,500원·2월 7,000원·합계 8,500원 정확.**

## 8g. 계정과목 선택 가이드 (0.32)

## 8h. 개인 가계부 모드 (0.33)

- [x] 사업자 정보가 없는 첫 실행 시 `개인 가계부로 시작` / `사업자로 시작` 두 카드가 뜨고, 다른 화면으로는 접근되지 않는다. **헤드리스 검증됨.**

- [x] 개인 모드 자산구입 유형: 예금/적금·가전/가구·차량이 선택되고 현금·보통예금 같은 배관용 계정은 뜨지 않는다. **헤드리스 검증됨.**
