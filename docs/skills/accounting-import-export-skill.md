# Accounting Ledger Import/Export Skill

이 문서는 회계장부 앱의 import/export를 다룰 때 적용하는 별도 스킬 문서다. 국세청 간편장부 Excel, 기존 엑셀 장부, CSV/XLSX, 카드 승인내역, 은행 입출금, 세금계산서, 현금영수증, PG 정산, 세무사 전달용 Excel/PDF 산출물을 만질 때 이 문서를 먼저 본다.

## 핵심 결정

| 항목 | 결정 |
|---|---|
| 공식 간편장부 포맷 | 국세청 간편장부 서식 호환 |
| 내부 원장 | 복식부기 기반 SSOT |
| 간편장부 Excel | 내부 원장의 import/export view |
| 국세청 프로그램 | v3.4 업무 흐름을 참조하되 매크로 구조는 따르지 않음 |
| 원본 보존 | 모든 import 원본과 행을 보존 |
| 확정 방식 | import 직후 확정 금지, 미리보기/매칭/사용자 확인 후 반영 |
| 통계 | 월별/분기별/반기별/연 합계 검증 |
| 세무사 전달 | 간편장부 Excel, 법정 신고서식 PDF/Excel, 증빙대장 모두 제공 |

## 공식 기준

국세청 간편장부 안내는 거래 발생일 순서로 수입, 비용, 사업용 유형자산 및 무형자산 증감을 기록하도록 안내한다. 국세청은 간편장부 Excel 서식을 제공하며, Excel 파일은 자동집계가 된다고 안내한다.

출처:

- 국세청 간편장부 안내: https://www.nts.go.kr/nts/cm/cntnts/cntntsView.do?cntntsId=7670&mi=2231
- 국세청 간편장부 서식: https://www.nts.go.kr/nts/cm/cntnts/cntntsView.do?cntntsId=7670&mi=2231

확인한 ZIP:

```text
간편장부 작성 프로그램(version 3.4)_250428.zip
```

구성:

|---|---|
| `간편장부 작성 프로그램(version 3.4) 사용자 설명서.pdf` | 공식 업무 흐름 참조 |
| `간편장부 작성 프로그램(version 3.4)_250428.xls` | Excel VBA 매크로 기반 프로그램. 실행 금지, 구조 참고만 |

설명서 기준 업무 흐름:

| 공식 메뉴 | 앱 기능 |
|---|---|
| 내 정보 관리 | 사업자/사업장/주업종코드 관리 |
| 거래처 관리 | 거래처 등록, 사업자등록번호 검증, 거래처 파일 업로드 |
| 개별 입력 | 간편장부식 거래 입력 |
| 파일 업로드 | 정해진 양식으로 장부 일괄 등록 |
| 수정/삭제/복사 입력 | 수정 이력, tombstone 삭제, 반복거래 복사 |
| 사업용 자산 입력 | 자산대장, 취득/매각/부대비용 |
| 감가상각비 입력 | 자산대장 기반 감가상각 계산, 연말 장부 반영 |
| 장부 출력 | 연도별 간편장부 |
| 영업현황표 | 거래유형별, 계정과목별, 기간별 집계 |
| 자산대장 | 자산 현황 출력 |
| 신고서식(소득세) | 간편장부소득금액계산서, 총수입금액 및 필요경비명세서, 감가상각비 조정명세서 |

반영할 주의사항:

1. 주업종코드는 계정과목 판단과 업종별 내용연수 기준에 영향을 주므로 필수 검증한다.
2. 거래처 사업자등록번호는 `-` 없이 10자리 입력을 허용하되, 저장 시 정규화한다.
3. 파일 업로드는 중복 입력 위험이 있으므로 원본 행 해시와 매칭키로 idempotency를 보장한다.
4. 업로드 결과는 성공/실패/수정필요를 행별로 표시한다.
5. 감가상각비는 임의 비용 입력이 아니라 자산대장 계산에서 연결한다.
6. 세무조정 항목은 합계만 저장하지 않고 조정내역을 별도 보관한다.

## 국세청 간편장부 열 매핑

| Excel 열 | 공식 항목 | 앱 내부 필드 |
|---|---|---|
| A | 월 | `transaction_month` 또는 `transaction_date` 파생값 |
| B | 일 | `transaction_day` 또는 `transaction_date` 파생값 |
| C | 계정과목 | `account_name`, `account_id`, `standard_account_code` |
| D | 거래내용 | `description` |
| E | 거래처 | `counterparty_name`, `counterparty_id` |
| F | 수입 금액 | `income_supply_amount` |
| G | 수입 부가세 | `income_vat_amount` |
| H | 비용 금액 | `expense_supply_amount` |
| I | 비용 부가세 | `expense_vat_amount` |
| J | 사업용 자산 금액 | `asset_amount` |
| K | 사업용 자산 부가세 | `asset_vat_amount` |
| L | 비고 | `evidence_hint`, `payment_hint`, `memo` |

## Import 규칙

1. `장부` 시트 4행 이후만 거래 후보로 읽는다.
2. 빈 행은 건너뛴다.
3. A/B가 월/일로 분리된 경우 장부 귀속연도와 결합한다.
4. A에 `1.5`처럼 입력된 경우 월/일로 파싱한다.
5. F/G가 있으면 수입 후보, H/I가 있으면 비용 후보, J/K가 있으면 자산거래 후보로 분류한다.
6. 두 구역 이상에 금액이 있으면 검토 필요로 표시한다.
7. `△`, 음수, 붉은색 자산 금액은 자산 매각/폐기 후보로 둔다.
8. L 비고의 약어를 증빙/결제 후보로 해석한다.
9. import 원본 파일 해시와 원본 행 JSON을 저장한다.
10. import 결과는 즉시 전표 확정하지 않고 사용자 확인 후 `source_transactions`에 반영한다.

비고 약어 후보:

| 값 | 후보 의미 |
|---|---|
| `세계` | 세금계산서 |
| `계` | 계산서 |
| `카드` | 신용/직불/선불카드 |
| `현영` | 현금영수증 |
| `영` | 기타 영수증 |
| `현금` | 현금 결제 |
| `외상` | 외상/미수/미지급 |

## Export 규칙

1. 세무사 전달용 기본 Excel은 국세청 간편장부 서식 호환으로 만든다.
2. 세무사 전달 패키지에는 법정 최신 신고서식 PDF/Excel도 함께 넣는다.
3. 증빙대장에는 이미지와 PDF 증빙의 Cloudinary 링크, 파일명, 해시, 거래 연결 정보를 넣는다.
4. 내부 복식 원장에서 `simple_book_rows` view를 만든 뒤 Excel에 쓴다.
5. 원천거래 1건이 여러 증빙/결제와 연결되어도 Excel에는 중복 손익으로 쓰지 않는다.
6. 자산 취득/매각은 비용 칸이 아니라 사업용 자산 칸으로 쓴다.
7. 일반과세자는 공급가액과 VAT를 분리한다.
8. 간이과세자와 면세사업자는 확정 세무유형에 맞춰 금액/VAT 칸을 조정한다.
9. `통계` 시트 합계와 내부 원장 합계를 비교한다.
10. 소득 종류 또는 사업장이 여러 개면 별도 장부/시트/파일로 구분한다.

## 검증 체크리스트

| 테스트 | 기대 결과 |
|---|---|
| 공식 서식 import | `장부` 4행 이후가 거래 후보로 읽힌다 |
| 월별 통계 | `통계` 시트 합계와 내부 월별 합계가 일치한다 |
| 수입/비용/자산 분리 | F/G, H/I, J/K가 각각 다른 거래 유형으로 매핑된다 |
| 비고 약어 | `세계`, `계`, `카드`, `현영`, `영`이 증빙 후보로 해석된다 |
| 2025 소급 | 장부 귀속연도 2025로 날짜가 생성된다 |
| 중복 방지 | 카드대금 출금이 신규 비용으로 들어가지 않는다 |
| 원본 보존 | import 파일 해시와 원본 행 JSON이 보존된다 |
| export 왕복 | 앱에서 export한 파일을 다시 import해도 거래 후보가 중복 확정되지 않는다 |
| 세무사 전달 패키지 | 간편장부 Excel, 법정 신고서식 PDF/Excel, 증빙대장이 함께 생성된다 |

## 금지사항

- 국세청 간편장부 Excel을 내부 SSOT로 삼지 않는다.
- Excel import 결과를 사용자 확인 없이 확정 전표로 만들지 않는다.
- 통계 시트 합계만 믿고 원장 검증을 생략하지 않는다.
- 월/일과 입력일을 혼동하지 않는다.
- 증빙 원본 없이 일일 매출 합계만 남길 때는 증빙 보관 경고를 빼지 않는다.
- 세무사 전달 시 간편장부 Excel만 만들고 법정 신고서식과 증빙대장을 누락하지 않는다.
