# Accounting Ledger Income Tax Reporting Skill

이 문서는 종합소득세 신고자료, 간편장부 신고서식, 복식부기의무자 경고, 기준경비율/단순경비율, 추계신고 검토를 다룰 때 적용하는 별도 스킬 문서다. 세무사 전달자료, 직접 신고 준비자료, 간편장부소득금액계산서, 총수입금액 및 필요경비명세서, 감가상각비 조정명세서, 기준경비율 추계신고 샘플을 만질 때 이 문서를 먼저 본다.

## 핵심 결정

| 항목 | 결정 |
|---|---|
| 기본 목표 | 장부 기장 기반 신고자료 생성 |
| 추계신고 | 예외/위험 검토 모드 |
| 복식부기의무자 | 간편장부/추계 선택 시 강한 경고 |
| 신고서식 | 국세청 공식 흐름에 맞춰 출력 |
| 법정 최신양식 | 국가법령정보센터 현행 별지서식 확인 후 출력 |
| 계산 근거 | 거래 판단메모와 legal_reference_checks에 연결 |
| 세무사 전달 | 국세청 간편장부 호환 Excel과 법정 최신 신고서식 PDF/Excel을 모두 출력 |

## 확인한 자료

| 파일 | 의미 |
|---|---|
| `간편장부 작성 프로그램(version 3.4) 사용자 설명서.pdf` | 간편장부 프로그램의 신고서식 출력 흐름 |
| `7.추계-기준경비율(복식부기의무자) 신고서 작성사례.pdf` | 복식부기의무자 기준경비율 추계신고 위험 사례 |

## 국세청 프로그램 신고서식 흐름

국세청 간편장부 작성 프로그램은 장부 작성 후 `신고서식(소득세)`에서 다음 서식을 확인/출력하도록 한다.

| 서식 | 앱 출력 |
|---|---|
| 간편장부소득금액계산서 | 수입/필요경비/세무조정/소득금액 |
| 총수입금액 및 필요경비명세서 | 장부상 수입금액과 필요경비 집계 |
| 감가상각비 조정명세서 | 사업용 자산 입력 시 생성 |

앱은 위 서식을 PDF/Excel 내보내기 대상으로 둔다.

## 세무사 전달 패키지

세무사에게 전달할 때는 두 양식을 모두 준비한다.

| 산출물 | 목적 |
|---|---|
| 국세청 간편장부 호환 Excel | 장부 원자료와 월별/계정별 검토 |
| 법정 최신 신고서식 PDF/Excel | 신고서식 기준 검토 |
| 증빙대장 | 이미지/PDF 증빙의 Cloudinary 링크와 파일 메타 확인 |
| 거래 판단메모 | 검토 필요 거래, 세무조정 후보, 미확정 쟁점 전달 |

## 법정 최신양식 원칙

리포트 기능은 법정 최신양식을 사용한다. 국세청 프로그램의 메뉴 흐름과 Excel 양식은 업무 호환 기준이고, 신고서식의 최신성은 국가법령정보센터 현행 시행규칙의 별지서식으로 확인한다.

확정 출력 조건:

1. `legal_form_snapshots`에 기준 서식이 저장되어 있어야 한다.
2. 리포트 초안은 `form_snapshot_id`를 가져야 한다.
3. 기준 서식에는 법령명, 법령ID, MST, 시행일, 별지서식 번호, 서식명, 개정일, 조회일이 포함되어야 한다.
4. 저장된 스냅샷보다 최신 서식이 발견되면 기존 리포트는 `requires_review`로 바꾼다.

2026-07-09 확인 예:

| 서식 | 확인값 |
|---|---|
| 간편장부소득금액계산서 | 소득세법 시행규칙 별지 제74호서식, `bylSeq=007400`, HWP |
| 총수입금액 및 필요경비명세서 | 별지 제74호서식 부표, 부표 개정 `2025-03-21` |

## 복식부기의무자 기준경비율 추계신고 샘플

샘플 핵심:

| 항목 | 값/의미 |
|---|---|
| 업종 | 한식음식점 |
| 업종코드 | `552101` |
| 신고유형 | 기준경비율 |
| 기장의무 | 복식부기의무자 |
| 주요경비 | 매입비용, 임차료, 인건비 |
| 정규증빙 미수취 | 증빙불비가산세 후보 |
| 장부 없음 | 무신고가산세와 장부 기록·보관 불성실 가산세 비교 |

이 샘플은 추계신고 구현을 장려하기 위한 것이 아니라, 장부의무 판정과 위험 경고를 위한 검증 자료다.

## 기준경비율 계산 흐름

```text
총수입금액
  -> 주요경비 차감
  -> 수입금액 * 기준경비율 차감
  -> 기준소득금액 계산
  -> 단순경비율 기반 비교소득금액 계산
  -> 둘 중 작은 금액 선택
  -> 소득공제/세액공제/가산세/기납부세액 반영
```

복식부기의무자 추계신고에서는 기타 경비에 기준경비율의 일부만 적용되는 경우가 있으므로, 법령/국세청 기준 출처를 반드시 확인한다.

## 앱 설계 원칙

1. 앱의 기본 목표는 장부 기장이다.
2. 추계/기준경비율은 보조 계산과 위험 경고 모드다.
3. 복식부기의무자 후보가 장부 없이 신고하려는 흐름을 선택하면 강한 경고를 표시한다.
4. 주요경비는 매입비용, 임차료, 인건비로 분리한다.
5. 정규증빙 미수취 거래는 가산세 검토 후보로 표시한다.
6. 영업손실보상금, 신용카드발행세액공제처럼 총수입금액 산입 검토가 필요한 항목을 별도 표시한다.
7. 신고서식 숫자는 내부 원장, 세무조정, 거래 판단메모의 추적 가능한 결과여야 한다.
8. 세무조정 항목은 합계만 저장하지 않고 세부 조정내역을 보관한다.

## 검증 체크리스트

| 테스트 | 기대 결과 |
|---|---|
| 복식부기의무자 판정 | 간편장부/추계 선택 시 위험 경고 |
| 기준경비율 샘플 | 주요경비, 기준소득금액, 비교소득금액 흐름 재현 가능 |
| 정규증빙 미수취 | 증빙불비가산세 검토 후보 표시 |
| 감가상각비 | 자산대장 기반 조정명세서 후보 생성 |
| 신고서식 출력 | 간편장부소득금액계산서, 총수입금액 및 필요경비명세서 출력 |
| 법정서식 스냅샷 | 리포트가 form_snapshot_id를 가지고 최신 서식 확인값과 연결된다 |
| 거래 판단메모 연결 | 세무조정 판단 근거가 메모로 남는다 |

## 금지사항

- 추계신고 계산을 장부 기장의 대체 목표로 만들지 않는다.
- 복식부기의무자 경고를 약하게 표시하지 않는다.
- 주요경비를 단일 비용 합계로 뭉개지 않는다.
- 세무조정 항목을 합계만 저장하고 근거를 버리지 않는다.
- 신고서식 숫자의 출처가 내부 원장/조정내역과 연결되지 않은 상태로 출력하지 않는다.
- 국가법령정보센터 최신 별지서식 확인 없이 확정 신고서식 리포트를 출력하지 않는다.
