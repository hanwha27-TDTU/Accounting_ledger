# Accounting Ledger Legal Basis Reference Skill (법령 기준 SSOT)

이 문서는 이 앱이 쓰는 **세무 법적 기준의 단일 출처(SSOT)** 다. 기장의무, 경비율 적용, 추계 소득금액 계산, 부가가치세 면세, 경비율·업종코드·간편장부 계정과목의 **조문·기준값·출처·앱 연동·목적(취지)** 을 한 곳에 모은다. 세무 기준이나 계산식을 만지기 전에 이 문서를 먼저 본다. 값은 여기서 읽고, 다른 문서·코드에 같은 숫자를 중복 정의하지 않는다(CLAUDE.md 하드룰: SSOT 중복 금지).

## 사용 규칙

- 이 표의 값은 **확정 신고 판단이 아니라 후보/기준**이다. 앱 표시는 `후보`·`검토용`·`개별 판단`으로 한다(하드룰).
- 값이 바뀌면 **이 문서 + 해당 앱 서비스 상수 + writer 스크립트 + 로직 테스트**를 같은 작업에서 함께 갱신한다.
- 모든 조문은 **법령 MCP(`search_law`/`get_law_text`) 원문 대조**로 확인하고, 아래 "확인 이력"에 대조일과 법령 MST를 남긴다. 추측으로 조문·숫자를 쓰지 않는다.
- 대분류→그룹 매핑은 **근사**다. 세분류 경계(부동산매매 vs 임대, 상품중개 등)는 개별 확인 대상임을 후보 표기로 고지한다.

## 1. 기장의무 — 간편장부 vs 복식부기

**근거**: 소득세법 제160조(장부의 비치ㆍ기록) 제1항~제3항, 소득세법 시행령 제208조 제5항 제2호.
**판정**: 직전 과세기간 수입금액이 아래 기준 **미만이면 간편장부대상자, 이상이면 복식부기의무자**. 신규개시자는 간편장부대상자(§208⑤1호).

| 그룹 | 업종(대분류 예시) | 기준(직전 수입금액) |
|---|---|---|
| 가목 | 농림어업, 광업, 도소매(상품중개 제외), 부동산매매, 그 밖에 | **3억원** |
| 나목 | 제조, 숙박·음식, 전기가스, 수도하수, 건설, 부동산개발공급(주거), 운수창고, 정보통신, 금융보험, 상품중개 | **1억 5천만원** |
| 다목 | 부동산임대, 부동산업, 전문·과학·기술, 사업시설관리, 교육, 보건, 예술·스포츠·여가, 협회·단체·수리·개인서비스, 가구내고용 | **7천 5백만원** |

**목적(취지)**: 세금은 정확한 소득 파악이 전제다. 규모가 큰 사업은 거래가 많고 복잡해 복식부기라야 정확하고, 작은 사업까지 강제하면 부담이 크다. 규모에 맞추되 정확성은 지키려는 제도.
**앱 연동**: `BookkeepingDuty`(index.html). 설정 업종코드 선택 시 기준 표시, 리포트 계산기에서 직전수입 입력 시 자동 판정.

## 2. 단순경비율 vs 기준경비율 적용

**근거**: 소득세법 시행령 제143조 제4항 제2호.
**판정**: 직전 과세기간 수입금액이 아래 기준 **미만이면 단순경비율, 이상이면 기준경비율**. 신규개시자는 단순경비율(§143④1호).

| 그룹 | 기준(직전 수입금액) |
|---|---|
| 가목 (도소매 등) | **6천만원** |
| 나목 (제조·음식·건설·정보통신·금융, **인적용역** 포함) | **3천 6백만원** |
| 다목 (전문서비스·교육·보건·예술·개인서비스 등) | **2천 4백만원** |

**⚠️ §208⑤(기장의무)와 그룹 경계가 다름**: §143④2호 나목은 "수리 및 기타 개인서비스업 중 부가가치세법 시행령 제42조제1호에 따른 **인적용역**"을 포함하고, 다목은 그 인적용역을 제외한다. 따라서 **인적용역(업종코드 94xxx)은 여기서 나목(3,600만)**, 기장의무(§208⑤)에서는 다목(7,500만)이다.
**목적(취지)**: 장부가 없어도 신고할 수 있게 업종 평균 경비율로 소득을 추정하되, 규모가 커지면 더 엄밀한 기준경비율로 넘긴다.
**앱 연동**: `ExpenseRateMethod`(index.html). 94xxx 나목 특례 처리.

## 3. 추계 소득금액 계산 (기준경비율)

**근거**: 소득세법 제80조 제3항, 소득세법 시행령 제143조 제3항. 비교소득금액 배율은 §143③1호 단서 + 소득세법 시행규칙(재정경제부령).

**소득금액 = min(① 기준소득금액, ② 비교소득금액)**

- ① 기준소득금액 = 수입금액 − 주요경비 − (수입금액 × **기준경비율** × [복식부기의무자 **½**, §143③1호 다목])
  - 주요경비 = 매입비용 + 사업용 고정자산 임차료(가목) + 종업원 급여·임금·퇴직급여(나목), **정규증빙** 필요
- ② 비교소득금액 = [수입금액 × (1 − **단순경비율**)] × 배율(**복식부기 3.4배 / 간편장부(단순경비율 적용대상) 2.8배**)
- 단순경비율 방법(§143③1호의2, 단순경비율 적용대상자만) = 수입금액 − (수입금액 × 단순경비율)

**공식 작성사례 검증** (국세청 「추계신고_기준경비율(복식부기의무자 한식점)」, 업종코드 552101):

| 항목 | 값 |
|---|---|
| 총수입금액 | 120,000,000 (매출 94,100 + 영업손실보상금 25,000 + 신용카드발행세액공제 900, 천원) |
| 주요경비 | 92,960,000 (매입 53,960 + 임차 16,000 + 인건 23,000) |
| 기준경비율 / 단순경비율 | 9.7% / 89.7% |
| ① 기준소득금액 | 120,000,000 − 92,960,000 − (120,000,000×9.7%×½) = **21,220,000** |
| ② 비교소득금액 | [120,000,000×(1−89.7%)]×3.4 = **42,024,000** |
| 소득금액 = min | **21,220,000** (필요경비 98,780,000) |

**복식부기의무자 추계 시 가산세**: 무신고가산세 max(수입금액×0.07%, 무신고납부세액×20%) **vs** 장부 기록·보관 불성실 가산세(산출세액×무기장비율×20%) 중 **큰 금액**. 증빙불비 2%(직전 수입 4,800만 이상). → 추계는 예외, 장부 기장이 원칙.
**목적(취지)**: 장부 없는 소규모 사업자도 신고 가능하게 하되, 원칙(장부)에서 벗어난 추계에는 가산세로 유도.
**앱 연동**: `EstimatedIncome.byStandardRate()`(index.html). 로직 테스트가 작성사례 수치로 고정.

## 8. 해외 구매·수입

**근거**: 부가가치세법 제4조(과세대상 = ①국내 공급 ②재화의 수입), 제52조(대리납부), 소득세법 제27조(필요경비).

| 구분 | 부가세 처리 | 근거 |
|---|---|---|
| 해외 물품 수입 | 세관이 수입부가세(10%) 부과 → 수입세금계산서. 과세사업자 매입세액 공제, 면세사업자 불공제(원가 포함) | 부가세법 §4 2호 |
| 해외 용역·구독(SaaS·클라우드·광고) | 국외사업자 용역 공급 → **대리납부**. 과세사업 사용은 제외되나 **면세사업자·비사업자는 10% 대리납부 의무** | 부가세법 §52 |
| 해외 현지 소비(출장 식대·교통) | 국내 부가세 없음 — 금액만 경비 | 소득세법 §27 |

**환율**: 외화는 거래일의 기준환율(매매기준율)로 원화 환산. **증빙**: 해외 지출은 국내 정규증빙 대상 아님 → 인보이스·카드전표로 갈음.
**앱 연동**: 가이드 `overseas` 토픽 + 용어 `수입부가세`·`대리납부`·`기준환율`. (자동 계산/입력 연동은 후속.)

| 1세대1주택 특례세율(시가표준액 9억원 이하 한정) | 표준세율 전 구간에서 0.05%p씩 인하(0.05/0.1/0.2/0.35%) | §111의2① (2021~2026년 한시) |
| 재산세 도시지역분(표준) | 과세표준의 0.14%(지자체 조례로 최대 0.23%까지 조정 가능, 도시지역 고시 지역만) | §112①2호·② |
| 지방교육세 | 재산세 본세(도시지역분 제외)의 20% | §151①6호 |
| 공유재산 안분 | 과세표준·세율은 항상 물건 전체 기준으로 계산하고, 그렇게 나온 세액만 지분권자별 지분율로 나눈다(과세표준 자체를 먼저 지분만큼 쪼개 세율 구간을 다시 매기지 않음) | §107①1호(공유재산은 그 지분에 해당하는 부분에 대해 그 지분권자가 납세의무), §107①2호(유사 사례인 건물·토지 소유자가 다른 경우도 "산출세액을... 안분계산"이라고 명시해 세액 사후분할 원칙을 뒷받침) |

**목적(취지)**: 부동산 보유에 매기는 지방세로, 보유세 형평성과 지방재정 확충이 목적. 1세대1주택 특례세율·2026년 한정 공정시장가액비율 인하는 실수요 1주택자의 세 부담을 완화하려는 정책 조정. 과세표준상한제(5%)는 공시가격이 급등해도 과세표준이 한 해에 너무 많이 뛰지 않도록 막는 완충장치(옛 세부담상한제와 취지는 같으나 산정 기준이 "재산세액 대비 %"에서 "과세표준 대비 %"로 바뀜).
**앱 연동**: `PropertyTax`(index.html) — `calculateHousing()`(`ownershipShare` 파라미터로 공유 지분율 반영). 계산 결과 아래 `propertyTaxStepsHtml()`이 계산 과정을 단계별 문장으로 보여준다(별도 계산 없이 `calculateHousing()`의 반환값만 읽어 서술 — 숫자가 어긋날 수 없는 구조). 공시가격 입력칸에 "어느 시점 기준 값을 넣어야 하는지" 안내(공동주택가격 공시 시기 규칙, 위 표). 나머지 과세대상(토지·건축물·선박·항공기)과 지역자원시설세(소방분)는 미구현(화면에 명시).
**공식 사례 대조 완료**: 사용자가 제공한 실제 서울시 이택스 재산세(주택1기분) 고지서(2026년 7월, 서초구, 동일지분 2인 공동명의 1세대1주택)로 검증했다. 처음엔 고지서의 "재산세과표 610,605,000원"을 (상한 미적용을 가정해) 시가표준액 1,356,900,000원으로 역산했는데, 나눗셈이 나머지 없이 딱 떨어진 게 우연이었다 — 사용자가 부동산공시가격 알리미 화면(공동주택가격 2026.1.1기준 **1,578,000,000원**, 2025.1.1기준 1,278,000,000원)을 제시해 실제 시가표준액이 다르다는 걸 확인했다. 다시 계산하면 raw 과세표준 = 1,578,000,000 × 45% = 710,100,000원으로, 고지서 과표(610,605,000원)보다 크다 — **과세표준상한액(시행령 §109의2, 직전연도+5%)이 실제로 발동**했다는 뜻이다. 상한 공식을 거꾸로 풀면 필요한 직전연도 과세표준상당액은 575,100,000원인데, 이는 2025년 공시가격(1,278,000,000원)에 같은 45% 비율을 곱한 값과 정확히 같다(2025년에도 유사한 1세대1주택 특례비율이 적용됐음을 시사하는 정황 — 2025년 법령을 직접 조회해 확정한 것은 아님). `calculateHousing({marketValue:1578000000, isOneHouseholdOneHouse:true, priorYearTaxBase:575100000, ownershipShare:0.5})`를 실행하면 과세표준이 고지서와 정확히 일치(610,605,000원)하고, 재산세 본세·도시지역분(전자송달 공제 전)·지방교육세가 고지서의 연간(1기×2) 실측값과 반올림 오차(10원 이하) 수준까지 일치한다. 처음의 "상한 미적용" 가정은 틀렸지만 우연히 같은 과세표준이 나와 최종 세액 자체는 처음부터 맞았다 — 다만 "왜 그 값이 나왔는지"의 설명이 틀렸었다. 로직 테스트를 실제 검증된 입력값(공시가격+직전연도 과세표준)으로 갱신해 오라클 테스트로 고정했다(`scripts/tests/logic.test.mjs`).
**남은 위험**: (1) 1세대1주택 판정(시행령 §110의2)은 다주택·상속·혼인·임대주택 등 12개 예외 호가 있어 앱이 자동 판정하지 않고 사용자 입력을 그대로 신뢰한다. (2) 지역자원시설세(소방분)·토지·건축물·선박·항공기는 여전히 미구현. (3) 실제 고지서 대조에서 확인된 800원 차이(전자송달·자동이체 공제)는 세법 계산이 아닌 지자체 행정 할인이라 의도적으로 미반영.

## 유지 규율 (이번 세션에서 확립한 교훈)

- **법령 원문이 SSOT**: 사용자 엑셀이 없어도 법령 MCP로 조문을 조회해 구현하고 근거를 명시한다.
- **작성사례 수치로 계산 엔진을 고정**: 공식 사례 수치를 로직 테스트에 박아 회귀를 막는다(예: 552101 → 21,220,000).
- **교차검증**: 같은 값이 여러 출처에서 일치하는지 확인한다(경비율 데이터 ↔ 작성사례).
- **인터랙티브 UI는 헤드리스 브라우저로 실제 구동 검증** 후 "확인함"이라 말한다(정직한 완료).
- **후보/개별판단 표기**: 세무 판정은 확정 신고값으로 표시하지 않는다.
- **눈높이 설명 동반**: 기준·근거뿐 아니라 목적·취지를 중학생 눈높이로 함께 제공한다(가이드 `.guide-why`).
